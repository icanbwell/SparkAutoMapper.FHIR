Exact¶

Contains¶

The search parameter returns resources that do not contain a match. From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

Not¶

The search parameter is processed as a string that searches text associated with the code/value - either CodeableConcept.text, Coding.display, or Identifier.type.text. From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

Text¶

The search parameter is a URI (relative or absolute) that identifies a value set, and the search parameter tests whether the coding is in the specified value set. From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

In¶

The search parameter is a URI (relative or absolute) that identifies a value set, and the search parameter tests whether the coding is not in the specified value set. From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

NotIn¶

Below¶

The search parameter tests whether the value in a resource subsumes the specified value (is-a, or hierarchical relationships). From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

Above¶

The search parameter only applies to the Resource Type specified as a modifier (e.g. the modifier is not actually :type, but :Patient etc.). From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

Type¶

The search parameter applies to the identifier on the resource, not the reference. From: http://hl7.org/fhir/search-modifier-code in valuesets.xml

Identifier¶

The search parameter has the format system|code|value, where the system and code refer to an Identifier.type.coding.system and .code, and match if any of the type codes match. All 3 parts must be present. From: http://hl7.org/fhir/search-modifier-code in valuesets.xml
